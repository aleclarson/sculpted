# Persistent Style Inspector for Preact and PandaCSS

## Core Concept

A packaged developer tool can provide an interactive style editing experience for a desktop app while persisting accepted changes back to source code. The package targets Preact, PandaCSS, and Vite with an in-app runtime inspector.

> Use Vite instrumentation and a browser-side runtime agent for inspection, selection, computed styles, highlighting, live preview, and source-aware persistence.

Browser APIs can identify selected DOM nodes and current computed CSS, but they cannot reliably map that CSS back to the correct `css({ ... })`, recipe call, responsive token, or semantic token in source. That source-aware mapping must be generated by a Vite instrumentation plugin and runtime metadata layer.

## Goals

1. Let a developer click an element in the running desktop app and inspect its styles.
2. Show the editable PandaCSS source model, not only generated CSS.
3. Support live preview without immediately changing source files.
4. Persist approved edits back to the original TS/TSX source.
5. Generate small, reviewable diffs.
6. Work with Preact, PandaCSS `css()`, and eventually Panda recipes.
7. Package the functionality as a reusable library for apps using the same stack.
8. Keep all filesystem write capabilities dev-only.

## Non-goals

1. Supporting arbitrary frameworks in the first version.
2. Supporting arbitrary CSS-in-JS libraries in the first version.
3. Replacing browser developer tools wholesale.
4. Making generated stylesheet edits the persistence mechanism.
5. Reliably rewriting all dynamic style expressions in the first version.
6. Shipping source path metadata or filesystem write access in production builds.

## MVP Scope

The first production-quality MVP should support Vite-only instrumentation, Preact TSX files, static Panda `css({ ... })` object literals, nested responsive/condition objects, in-app element selection, computed style display, runtime preview stylesheets, AST-based patching, diff-before-save, and dev-only production stripping.

Defer variable style object resolution, full recipe editing, compound variants, computed-CSS-to-token conversion, visual drag controls, multi-file refactors, Preact developer-tool internals, and non-Panda CSS support.

## Final Recommendation

Design the package around four separable layers: Vite instrumentation, runtime agent, inspector UI, and AST patcher. The most important design decision is to avoid making generated CSS the source of truth. Runtime inspection is useful for selection and preview, but Panda source code is the only correct persistence target.
